# BLAS Implementations Performance Comparison

This repository provides a concrete codebase for evaluating and comparing the performance of different BLAS (Basic Linear Algebra Subprograms) implementations.

The primary benchmark demonstrates the execution time required to solve a linear system $Ax = b$ using the **Steepest Descent** algorithm on a generated Symmetric Positive Definite (SPD) matrix.

## Test Environment

- **CPU**: Intel Core i5-14400
- **GPU**: NVIDIA GeForce RTX 4060 Ti
- **OS**: Windows 11 with WSL2 (Ubuntu 24.04.4 LTS)

## Tested Implementations

The project currently tests and supports the following BLAS implementations via CMake configurations:
- **Netlib CBLAS**: The original reference implementation without hardware-specific optimizations.
- **OpenBLAS**: An optimized, multi-threaded open-source implementation.
- **Intel oneMKL**: Intel's Math Kernel Library, heavily optimized for modern x86 architectures.
- **OpenBLAS (Single Thread)**: OpenBLAS forced to use a single thread (using the pure serial version).
- **Intel oneMKL (Single Thread)**: oneMKL linked against the sequential library.
- **cuBLAS (Hybrid)**: GPU-accelerated steepest descent using NVIDIA cuBLAS, while matrix generation remains on the CPU (using OpenBLAS).
- **cuBLAS (Optimized)**: A fully GPU-accelerated pipeline where SPD matrix generation (via cuRAND and custom kernels) and the steepest descent algorithm are entirely executed on the Device memory to eliminate PCIe bandwidth bottlenecks.

## Build \& Run

The project is built using CMake and Ninja. Ensure you have the necessary BLAS libraries, CUDA toolkit, and compilers installed on your system.

```bash
# Install Ninja if you don't have it (Ubuntu/Debian)
sudo apt-get install ninja-build

# Configure and build
cmake -DCMAKE_BUILD_TYPE=Release -G Ninja -S . -B cmake-build-release
cmake --build cmake-build-release
```

### Usage
You can execute the individually generated targets with the following command-line signature:
```bash
./cmake-build-release/main_<implementation> [n] [filename] [verbose]
```

**Parameters:**
- `n`: Matrix dimension size (Default: `10`)
- `filename`: The file path to save the generated linear system $A$, $b$, and solution $x$ in 17-decimal precision (Default: `./result.txt`)
- `verbose`: Pass `1` to print the generated matrix and vectors to stdout, `0` to keep it silent (Default: `0`)

**Example:**
```bash
./cmake-build-release/main_optimize 4000
```

## Benchmark Script

This repository includes a Python script (`benchmark.py`) to automatically evaluate all 7 BLAS implementations across various matrix sizes (100, 500, 1000, 4000) and generate a 4x4 comparison grid plot.

**Prerequisites:**
1. **Build the project first:** Ensure you have successfully compiled all executables into the `cmake-build-release` directory (see the [Build & Run](#build--run) section).
2. **Install Python dependencies:** Use `uv` to sync the environment dependencies:
   ```bash
   uv sync
   ```

**Usage:**
Execute the benchmark script:
```bash
uv run python benchmark.py
```
After successful execution, the script will generate:
- `benchmark.log`: The raw execution outputs and timings for all tested configurations.
- `benchmark_plot.png`: A 4x4 visual grid plotting various performance metrics (SPD Generation Time, Vector Generation Time, Algorithm Execution Time, and Normalized Execution Time per Iteration).

## Secondary Utilities (Python Script)
This repository also contains `scipy_solver.py`, a script heavily utilizing Python's `scipy.linalg` optimizations. 
This script reads the saved linear system generated by the C/C++ targets (`result.txt`) and independently solves it to verify the algorithmic correctness and solution precision of our C/CUDA Steepest Descent implementations.

**Prerequisites:**
1. **Build and run the project first:** Ensure you have compiled the executables and generated a data dump file (e.g., `result.txt`) by executing one of the BLAS targets.
2. **Install Python dependencies:** Use `uv` to sync the environment dependencies:
   ```bash
   uv sync
   ```

**Usage:**
```bash
uv run python scipy_solver.py -f <filename>
```
**Options:**
- `-f` / `--file`: The path to the data dump file generated by the executable. (Required, e.g., `-f ./result.txt`)
